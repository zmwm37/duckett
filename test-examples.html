Lists

Lists come in three types, ordered, unordered, and definition.

Ordered lists have numbers with them, like instructions.

  1. Lather
  2. Rinse
  3. Repeat
In constrast, unordered lists can appear as bulleted lists:
  • one fish
  • two fish
  • red fish
  • blue fish
You can use attributes to sepcify type of bullet, but it's better to use CSS formatting to accomplish this. There are also definition lists and nested lists
Definition list
A type of list with a structure useful for showing definitions. The term is listed and left-adjusted, while the definition is indented below it. Use dt tags for the term and dd tags for the definition.
Nested list
Lists within lists, either ordered or unordered. (No example shown )

Linking

Linking uses the a tag and the "href" attribute to specify the link. If a link it exerternal, then it needs an absolute URL, but if the link is to the a different page of the same website, then a relative URL will suffice.

When organizing the directory, each main page of the website should have its own dedicated folder. This helps with organization. The name of the main page within each section should be index.html because this is what web browsers will return by default if no webpage is specified.

To link to parts of the same page, use the id attribute of the element you want to link to, and put it in the href attribute of the link

Images

Images can be from jpg, png, or gif files

  • jpg: good for complex images with many colors
  • png: good for simple images with few colors
  • gif: similar to jpg, also can be a series of images to make simple animations

Images have an "alt" attribute that should always be populated with good description because it is used by web readers to describe pictures to visually impaired.

Images should be saved as the size that they will be on the webpage
